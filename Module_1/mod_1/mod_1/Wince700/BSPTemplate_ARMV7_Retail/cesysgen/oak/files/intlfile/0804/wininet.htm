证书概述

证书是数字凭证，其中包含与表示该凭证的设备相关的信息。例如，Web 服务器设备中的证书包含有关该设备自身的信息。有效的证书由证书颁发机构(CA)签发，其中包含下列信息：证书颁发者、公钥以及过期日期。

当您连接到 Internet 时，您访问的网站的证书的可信性至关重要。不可信任的或未正确验证的网站可能会危害您设备上存储的信息。如果不确定，您可以通过您的设备查看该 Web 服务器证书，然后再决定是否访问该网站。

您设备上的安全和隐私功能有助于保护您的隐私，并确保您个人信息的安全。这些隐私功能使您可以保护自己的个人识别信息(PII)。而且，还可以帮助您控制您访问的网站以何种方式使用您的 PII。通过设备上的隐私控制，还可以指定隐私设置来确定在访问网站时共享哪些信息，以及哪些信息可以保存到您的设备上。

客户端证书

客户端证书是一种数字声明，用于验证用户或设备（如个人计算机）的标识。颁发客户证书是为了建立标识，并为使用各种设备交换的信息提供一种验证和安全级别。某些网站要求您使用客户证书标识您自己。当您向另一台计算机发送客户证书时，使用的是允许您以安全方式与该设备共享数据的公钥和私钥。公钥和私钥还可以阻止未授权用户访问您的设备以及其中的信息。

下表显示了证书中包含的信息：

  • 接受方的公钥值和标识符信息
  • 颁发者的标识符信息
  • 有效期
  • 颁发者的数字签名

主体是接收证书的实体；证书颁发者是证书颁发机构。

个人证书和根证书

您可以在自己的设备上存储两类证书：为您创建标识的个人证书和为 Web 服务器创建标识的根证书。例如，通过 Internet 发送个人信息时，个人证书向您访问的网站验证您；而根证书在您访问网站时帮助保护您的个人信息。

隐私概述

使用您设备上的隐私功能，可以保护您的个人识别信息 (PII)。隐私功能帮助您控制您访问的网站如何使用您的 PII。这些功能使您可以通过指定隐私设置，来确定是否允许网站将信息（如 cookie）保存到您的个人计算机中。

Cookie

每次访问网站时，该网站都会在您的个人计算机上存储一个称为 cookie 的小文件。网站创建 Cookie 是为了存储信息，例如您访问该网站时所使用的一些首选设置。一些 cookie 包含您在访问的网站中查看过的网页的记录，以便在您下次访问该网站时帮助您自定义视图。

Cookie 也可以存储个人识别信息，例如您的姓名、电子邮件地址、家庭地址、单位地址或电话号码。但是，网站仅具有您所提供的个人识别信息的访问权限。

下表显示了您设备上的隐私功能：

  • 隐私设置，指定您的计算机如何处理 cookie
  • 隐私警报，在您试图访问不满足您隐私设置条件的网站时通知您
  • 查看网站的 P3P 隐私策略的能力

您可以使用您设备上的隐私功能限制 cookie 的使用。但是，由于许多应用程序需要 cookie 中包含的信息，因此完全禁用 cookie 并不总是最好的解决方案。

第一方与第三方内容

第一方内容包含有关主域或您正在访问的网站的信息。第三方内容来源于其他域。例如，当您访问包含横幅广告的网站时，您正在访问的网站被认为是第一方内容，而横幅广告被认为是第三方内容。

许多商业网页使用第一方和第三方 cookie 的组合来收集和中继有关这些页面的数据。您设备上的隐私功能可以分辨第一方和第三方内容，并分别与第一方和第三方建立关系。事实上，用于第三方内容的默认隐私设置比用于第一方内容的更严格。基于 Windows Embedded Compact 的设备上的隐私设置可以禁用第三方 cookie，并可以启用第一方 cookie，或者提示您启用或禁用第一方 cookie。

安全概述

您设备上的安全功能可以禁止他人访问未授权其访问的信息，例如您在 Internet 上购物时输入的信用卡信息。安全功能还可以保护您的个人计算机或无线设备免受不安全软件的危害。

无论何时您访问使用安全套接字层的超文本传输协议(HTTPS)的网站，都可以安全地浏览。HTTPS 是安全的 Web 通信协议，它对通过此连接发送的数据进行加密。第三方无法截获和破解使用 HTTPS 加密的数据。

安全警报

出现下列情况时，会显示安全警报：

  • 您要连接的 Web 服务器包含的服务器证书无效
  • 您要连接的 Web 服务器包含混合型安全（安全和不安全的数据）
  • 您正在离开安全连接
