인증서 개요

인증서란 증명서를 나타내는 장치에 대한 정보가 들어 있는 디지털 증명서입니다. 예를 들어, 웹 서버 등의 장치에는 자체 정보를 포함하는 인증서가 있습니다. 유효한 인증서는 인증 기관(CA)에서 발급한 것으로, 이 인증서에는 인증서를 발급한 사람, 공개 키, 만료 일자 등에 대한 정보가 있습니다.

인터넷에 연결하는 경우 사용자는 방문하는 웹 사이트의 인증서를 신뢰할 수 있어야 합니다. 신뢰할 수 없거나 올바르게 인증되지 않은 웹 사이트는 사용자 장치에 저장된 정보를 손상시킬 수 있습니다. 확실하지 않은 경우 장치에서 웹 서버 인증서를 본 후 웹 사이트를 방문할지 여부를 결정할 수 있습니다.

사용자 장치의 보안 및 개인 정보 기능은 개인 정보를 보호할 수 있게 도와주며 개인 정보가 안전한지에 대해서도 확인할 수 있습니다. 이러한 개인 정보 기능을 통해 개인 식별 정보(PII)를 보호할 수 있습니다. 또한 방문하는 웹 사이트가 사용자의 PII를 어떻게 사용할 수 있는지를 제어할 수 있게 도와줍니다. 사용자 장치의 개인 정보 컨트롤을 사용하여 사용자가 웹 사이트를 방문했을 때 어떤 정보를 공유할지, 사용자 장치에 어떤 정보가 저장될지를 결정하는 개인 정보 설정을 지정할 수도 있습니다.

클라이언트 인증서

클라이언트 인증서란 사용자 또는 개인용 컴퓨터와 같은 장치의 식별 정보를 확인하는 디지털 문서입니다. 클라이언트 인증서는 ID를 설정하고 여러 장치를 사용하여 교환된 정보의 보안 및 인증 수준을 제공합니다. 일부 웹 사이트에서는 클라이언트 인증서를 사용하여 사용자를 식별하도록 요청합니다. 다른 컴퓨터에 클라이언트 인증서를 보낼 경우 공개 키 및 개인 키를 사용하여 해당 장치와 안전하게 데이터를 공유할 수 있게 합니다. 또한 공개 키 및 개인 키는 권한 없는 사용자가 사용자의 장치 및 해당 정보를 액세스하지 못하도록 도와줍니다.

인증서에 포함된 정보는 다음과 같습니다.

  • 주체의 공개 키 값 및 ID 정보
  • 발급자의 ID 정보
  • 유효 기간
  • 발급자의 디지털 서명

주체는 인증서를 받는 대상이며, 인증서의 발급자는 인증 기관입니다.

개인 및 루트 인증서

장치에는 사용자의 ID를 설정하는 개인 인증서와 웹 서버의 ID를 설정하는 루트 인증서 두 가지를 저장할 수 있습니다. 예를 들어, 인터넷을 통해 개인 정보를 보낼 경우 개인 인증서는 사용자가 방문하는 웹 사이트에서 사용자를 인증하며, 루트 인증서는 사용자가 웹 사이트를 방문하는 동안 사용자의 개인 정보를 보호할 수 있게 도와줍니다.

개인 정보 개요

장치의 개인 정보 기능을 사용하여 개인 식별 정보(PII)를 보호할 수 있습니다. 또한 방문하는 웹 사이트가 사용자의 PII를 어떻게 사용하는지를 제어할 수 있게 도와줍니다. 이 기능을 사용하여 웹 사이트에서 사용자 컴퓨터에 쿠키 등의 정보를 저장하도록 할 것인지 여부를 결정하는 개인 정보 설정을 지정할 수 있습니다.

쿠키

웹 사이트를 방문할 때마다 웹 사이트는 사용자의 개인용 컴퓨터에 쿠키라는 작은 파일을 저장합니다. 쿠키는 웹 사이트 방문 시 사용자 기본 설정 같은 정보를 저장하기 위해 웹 사이트에서 만듭니다. 일부 쿠키에는 방문한 웹 사이트에서 사용자가 본 웹 페이지의 기록도 들어 있어, 다음에 방문할 때 웹 사이트 보기를 사용자 정의할 수 있도록 도와줍니다.

또한 쿠키는 사용자 이름, 전자 메일 주소, 집 또는 회사 주소나 전화 번호 등의 PII도 저장할 수 있습니다. 하지만 웹 사이트는 사용자가 제공하는 PII에만 액세스할 수 있습니다.

사용자 장치의 개인 정보 기능은 다음과 같습니다.

  • 컴퓨터가 쿠키를 처리하는 방법을 지정하는 개인 정보 설정
  • 개인 정보 설정의 조건과 일치하지 않는 사이트에 액세스하려는 경우 알아두어야 하는 개인 정보 경고
  • 웹 사이트의 P3P 개인 정보 정책을 보는 기능

사용자 장치의 개인 정보 기능을 사용하여 쿠키 사용을 제한할 수 있습니다. 하지만 많은 응용 프로그램이 쿠키에 있는 정보에 의존하기 때문에, 쿠키를 모두 사용할 수 없게 하는 것이 항상 최선의 해결책은 아닙니다.

자사 콘텐트와 타사 콘텐트

자사 콘텐트에는 호스트 도메인 또는 사용자가 방문 중인 웹 사이트에 관련된 정보가 들어 있습니다. 타사 콘텐트는 다른 도메인과 관련되어 있습니다. 예를 들어 사용자가 배너 광고가 들어 있는 웹 사이트를 방문하는 경우, 방문 중인 해당 웹 사이트는 자사 콘텐트로, 배너 광고는 타사 콘텐트로 간주됩니다.

많은 상업용 웹 페이지에서는 해당 페이지에 관련된 데이터를 수집하고 릴레이하기 위해 자사 쿠키와 타사 쿠키를 함께 사용합니다. 사용자 장치의 개인 정보 기능은 자사 콘텐트와 타사 콘텐트를 구별하며, 자사 콘텐트에는 타사 콘텐트와 다른 관계를 설정합니다. 사실 타사 콘텐트에 대한 기본 개인 정보 설정은 자사 콘텐트보다 더 엄격합니다. Windows Embedded Compact 기반 장치에 개인 정보 설정을 사용하면 타사의 쿠키를 사용할 수 없도록 해주며, 자사 쿠키를 허용하거나 이에 대한 허용 여부를 선택할 수 있도록 해줍니다.

보안 개요

사용자 장치의 보안 기능은 인터넷에서 쇼핑할 때 신용 카드 정보 같이 사용자가 액세스 권한을 제공하지 않은 정보에 타인이 액세스하지 못하도록 도와줍니다. 또한 보안 기능은 안전하지 않은 소프트웨어로부터 사용자의 개인용 컴퓨터나 무선 장치를 보호해 줍니다.

또한 HTTPS를 사용하는 웹 사이트를 방문할 때도 항상 안전하게 탐색할 수 있게 해줍니다. HTTPS는 이 연결을 통해 전송되는 데이터를 암호화하는 안전한 웹 통신 프로토콜입니다. HTTPS를 사용하여 데이터를 암호화하면 중간에 다른 사람이 끼어들어 그 내용을 알아낼 수 없게 됩니다.

보안 경고

보안 경고는 다음과 같은 경우에 나타날 수 있습니다.

  • 연결 중인 웹 서버에 잘못된 서버 인증서가 포함되어 있을 때
  • 연결 중인 웹 서버에 안전한 데이터와 안전하지 않은 데이터가 섞여 있을 때
  • 보안 연결을 끊을 때
